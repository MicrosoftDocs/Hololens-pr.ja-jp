# <a name="share-your-hololens-with-multiple-people"></a>複数のユーザー HoloLens共有する

## <a name="overview"></a>概要

多くの場合、企業は多くの共有デバイスHoloLensしています。 個々の要件HoloLensに応じて、アプリケーションを使用する方法は、ボード全体で柔軟に使用できます。 マルチユーザー エクスペリエンスの例を次に示します。

- HoloLens 2 用の Windows Autopilot を使用して、HoloLens 2 デバイスのフリートが設定されます。各デバイス上の会社のアプリケーションのポートフォリオは一貫しています。 さまざまなキオスク プロファイルを設定し、異なるキオスク プロファイルとグループAzure ADしました。 各ユーザーは、FIDO2 キー HoloLensして独自の Azure AD アカウントにサインインして、カスタマイズされたエクスペリエンスを提供することで、Azure AD にログインします。
- 独立系ソフトウェア ベンダー (ISV) は、D365 HoloLens 2 を使用した Remote Assist デバイスとその事業ライン (LOB) アプリケーションを顧客の会社にレンタルします。 これらのデバイスは、LOB アプリとデバイスのみを含むキオスクRemote Assist構成され、複数のエンド ユーザー間で共有されます。 WDAC は、アプリの起動を設定し、Microsoft Edgeするために使用されます。 レンタルに含まれる USB-C バッテリ パックは、デバイスを複数のシフトでフル 充電状態に保つ場合に使用します。
- 企業のエンド ユーザーは、新しいデバイスに接続できるよう、デバイス上の Bluetooth に対して調整を試みるが、Page 設定 Visibility ポリシーが有効になっていると、[デバイス] ページが表示されるのを制限できます。 必要に応じて他のページ (Wi-Fi など) へのアクセスが引き続き許可され、同じ HoloLens を持つ複数の場所で Remote Assist を使用できます。

### <a name="identity-and-authentication"></a>[ID と認証](hololens-identity.md)

1 つのデバイスに複数のアカウントを持つことを計画している場合は、すべての認証モードAzure ADアカウントを使用できます。 これらの認証方法は、Iris キーや FIDO2[キー Windows Hello](/windows-hardware/design/device-experiences/windows-hello)の認証方法に基づいて行います。

- FIDO 2 セキュリティ キーは、複数のデバイス、多くのユーザー、または新しいデバイスを常に使用している場合に優れています。
- ユーザーが 10 人以下の場合、Iris は、以前に同じデバイスにサインインしたユーザーをサインインする高速なソリューションです。

### <a name="device-management"></a>[デバイス管理](hololens-csp-policy-overview.md)

デバイスがユーザー間で共有されている場合は、デバイスの制限を使用する必要があります。 デバイス管理を使用すると、一部のポリシーを設定して、ユーザーがデバイスを使用したり、更新プログラムを管理したり、デバイスで実行できる操作を制限したりすることができます。 一般的なデバイスの制限 [を](hololens-common-device-restrictions.md)確認し、これらの推奨事項が組織に合うように見える場合は確認してください。 使用するポリシーがわかっている場合は、Microsoft の エンドポイント マネージャー [(MDM)](hololens-mdm-configure.md)またはプロビジョニング パッケージを使用して適用できます。

### <a name="advanced-device-management---kiosk-and-wdac"></a>高度なデバイス管理 -[キオスク](hololens-kiosk.md)[と WDAC](windows-defender-application-control-wdac.md)

場合によっては、エンド ユーザーがアクセスできるアプリケーションを制限することができます。 キオスク モード を使用して、スタート メニューでユーザーに表示されるアプリを [制限できます](hololens-kiosk.md)。 キオスクは、ユーザー、Azure グループ、または特別なユーザーの種類に基づいて異なるスタート メニューを表示するように構成できます。このような訪問者、またはデバイス所有者を除外する。 複数のアプリを選択するか、単一のアプリを選択できます。 マルチ アプリ キオスクでは、あるアプリが別のアプリの起動を停止しないので、ストアまたは別のアプリが利用可能な場合でも、ユーザーは別のアプリを起動できます。

また、Windows Defender [Application Control (WDAC)](windows-defender-application-control-wdac.md)を使用してアプリまたはサービスの起動を完全に停止して、アプリを制限することもできます。 WDAC はキオスクとは異なります。これは、HoloLens の UI は変更されませんが、ブロックされたアプリの起動は許可されません。

[ページ 設定可視性は](settings-uri-list.md)、デバイスに制限を追加するもう 1 つの方法です。 場合は、設定 アプリの一部のページへのアクセス権をユーザーに付与する必要がありますが、ページ 設定 Visibility を使用してアクセスを制限できるすべてではありません。 これは、たとえば、ユーザーが Wi-Fi を変更する必要があるが、ユーザーが [アカウント] ページにアクセスしたくない場合に便利です。

### <a name="physical-management"></a>物理管理

複数のユーザー間でデバイスを共有する場合は、いくつかの物理的な考慮事項があります。

- デバイスがシフト間で課金を行う。
- シフトにデバイスが必要で、複数のシフトを続けする必要がある場合は、シフトの開始時に外部バッテリを使用し、デバイスの熱方向の管理に従って大幅な料金が引き続き発生する場合を [検討](hololens2-charging.md#managing-heat)してください。
- デバイスを格納する場合、デバイスは接続された状態を維持し、ネットワークに接続します。 これは、OS とアプリの更新を確実に行う最善の方法です。
- ユーザー間でデバイスを [クリーンアップする方法を](hololens2-maintenance.md) 検討します。
- 1 人のユーザーに共有 PIN/パスワードを使用している場合、1 人の共有ユーザーを持つデバイスの場合は、デバイスの側に PIN/パスワードを配置しない。
- 1 つの共有ユーザーを持つ複数のデバイスの場合は、さまざまな PIN/パスワードを使用します。

## <a name="share-with-multiple-people-each-using-their-own-account"></a>それぞれ独自のアカウントを使用して複数のユーザーと共有する

個々Azure Active Directory (Azure AD) アカウントは、ユーザーに推奨される最も安全な ID HoloLens 2です。 独自のアカウントを使用Azure AD、複数のユーザーがデバイスに独自のユーザー設定とユーザー データを保持できます。 一度にサインインできるユーザーは 1 人のみです。 ユーザーがサインインすると、HoloLensユーザーがサインアウトします。

複数のユーザーが自分のアカウントを自分のアカウントで使用HoloLensするには、次の手順に従って構成します。

1. デバイスを [設定するときに、[](hololens2-start.md)マイ **ワークまたは** 学校が所有している] を選択し、アカウントを使用してサインインAzure ADします。
1. セットアップが完了したら、アカウント設定 ([アカウント] ) に [設定 >ユーザー] が含 **まれる必要があります**。

> [!NOTE]
> デバイスがアカウントを使用して設定されていないAzure ADリセットまたは再フラッシュして、 [適切に設定](hololens-recovery.md) する必要があります。

このオプションをHoloLens、各ユーザーは次の手順に従います。

1. 別のユーザーがデバイスを使用している場合は、次のいずれかのオプションを選択します。
   - 電源ボタンを 1 回押してスタンバイ状態に戻り、電源ボタンを再度押してロック画面に戻ります
   - 現在のユーザーをサインアウトするには **、スタート メニュー** ユーザー タイルを選択するか、[**電源**] メニューからサインアウトを選択します。

1. 自分のAzure AD資格情報を使用してデバイスにサインインします。  
    - デバイスを初めて使用する場合は、自分の目に合HoloLens調整する必要[](hololens-calibration.md)があります。
    - 以前にデバイスを使用した場合:
        - [Windowsバージョン 20H2 ビルド 19041.1128](hololens-release-notes.md#windows-holographic-version-20h2)以上の Holographic では、ディスプレイは品質と快適な表示エクスペリエンスに合わせてシームレスに調整されます。
        - 以前のビルドでは、目に合わせて調整するために手動調整が必要です。

> [!TIP]
> ユーザーがまだデバイスにサインインしていない場合は、次の 2 つの方法のいずれかを試して、ログインを高速化します。
>
> - **FIDO 2 セキュリティ** キー: FIDO2 セキュリティ キーが自動的に認識され、ユーザーはユーザーの資格情報を入力したり、MFA を使用したりする必要が生じしません。 これは、新しいデバイスでサインインする最も高速な方法です。
> - **Web** 認証: 新しいデバイスにサインインするときに、別のデバイスから[サインイン] リンクを選択すると [、aka.ms/devicelogin](https://login.microsoftonline.com/common/oauth2/deviceauth)で使用できる 9 文字のコードを生成して、デバイスのユーザーとしてサインインするか、キーボードを使用してユーザー名とパスワードを入力できます。

デバイス ユーザーの一覧を表示したり、デバイスからユーザーを削除したりするには、[アカウント] **[** その他のユーザー] の設定  >  **に**  >  **移動します**。

## <a name="share-with-multiple-people-all-using-the-same-account"></a>同じアカウントを使用して複数のユーザーと共有する

複数のユーザーが 1 つのユーザー アカウントHoloLensデバイスを共有することもできます。 HoloLens ユーザーが個々の ID (Azure AD アカウント) を使用してデバイスにログインする場合は推奨ですが、これは一部の組織では使用できません。

次の 2 つの共有デバイスメソッドを使用できます。

- **1 つのデバイスを共有** する複数のエンド ユーザー - HoloLensデバイスは、任意の従業員がデバイスを使用できる指定されたスペースに割り当てされます。 たとえば、クリーン ルームや医療スイートなどです。

- **複数のデバイスを共有** する複数のエンド ユーザー - HoloLensは、従業員が任意のデバイスを使用できる共有記憶域スペース内に置かされます。 たとえば、石油リグや自動車販売店/駐車場などです。

同じアカウントのサインインを維持しながら、新しいユーザーが初めてデバイスにアクセスすると、表示エクスペリエンスをすばやく調整してカスタマイズするようにユーザーに求めるメッセージが表示されます。 デバイスには、調整情報が格納され、各ユーザーの表示エクスペリエンスの品質と快適性が自動的に最適化されます。 ユーザーはデバイスを再度調整する必要はありません。
